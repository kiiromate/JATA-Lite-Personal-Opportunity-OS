# Bulk Opportunity Triage

JATA Lite v0.3 is for fast assessment and pipeline tracking across many opportunities. It prepares Kaze to focus on the best options first. It does not submit applications.

## 1. Import A CSV

```bash
pnpm start import fixtures/bulk-opportunities.csv
```

Required columns:

- `company`
- `role`
- `description`

Optional columns:

- `url`
- `source`
- `deadline`
- `method`
- `contact`
- `notes`
- `salary`
- `location`
- `remote`

The importer validates every row. Invalid rows are skipped and reported. Duplicates are skipped unless `--update` is passed.

## 2. Bulk Score

```bash
pnpm start score --all
pnpm start score --status new
pnpm start score --limit 25
```

Scoring is deterministic and local. It adds:

- `priorityBand`
- `effortEstimate`
- `applicationRiskLevel`
- `recommendedAction`

## 3. Generate A Shortlist

```bash
pnpm start shortlist
pnpm start shortlist --top 10
pnpm start shortlist --band A
pnpm start shortlist --max-risk medium
```

Outputs:

- `outputs/shortlist.md`
- `outputs/shortlist.csv`

Use the shortlist to choose where to spend preparation time. A-band items should be reviewed first. B-band items need judgment. C-band items are usually parked. D-band items should be ignored unless new context changes the fit.

## 4. Batch Generate Packs

```bash
pnpm start generate-batch --top 5
pnpm start generate-batch --band A
pnpm start generate-batch --ids id1,id2,id3
```

The default batch limit is 5. D-band packs are not generated by default. To generate a D-band pack, Kaze must explicitly pass its ID.

Every pack remains human-review only. Kaze must verify claims and manually submit outside JATA Lite.

## 5. Track Pipeline Status

```bash
pnpm start status <opportunityId> <status>
pnpm start next <opportunityId> "next action text"
pnpm start applied <opportunityId>
pnpm start followup <opportunityId> <YYYY-MM-DD>
pnpm start ignore <opportunityId>
```

`pnpm start applied <opportunityId>` records that Kaze manually applied outside the tool. It does not submit anything.

`pnpm start followup <opportunityId> <YYYY-MM-DD>` records a local follow-up date for the daily brief.

## 6. Daily Queue

```bash
pnpm start brief
```

The daily brief shows:

- today's top 5 opportunities
- urgent deadlines
- review-ready packs
- follow-ups due
- stale opportunities with no action for 7+ days
- the 20-Minute Workflow
- low-priority items not to touch today

## 7. Export Tracker

```bash
pnpm start export
```

The tracker CSV includes priority, risk, effort, recommendation, applied timestamp, follow-up date, last update, and pack path.

## Bulk Preparation Versus Auto-Application

Bulk preparation means importing, scoring, ranking, drafting, and tracking locally.

Auto-application would mean submitting, sending, replying, or interacting with external services automatically.

JATA Lite only does bulk preparation. It never submits applications.
